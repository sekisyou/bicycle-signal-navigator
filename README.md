# Bicycle Signal Navigator

自転車利用者向け信号予測ナビアプリ

## 背景・目的
大学まで片道27kmを自転車で通学する中で、
信号待ちによる時間ロスやストレスを感じたことをきっかけに開発しました。

## 主な機能
- ルート検索
- 信号通過予測
- GPS記録
- 走行ログ保存

## 使用技術
- React / JavaScript
- HTML / CSS
- Leaflet / OpenStreetMap

## 工夫した点
- 実走データを用いた予測精度改善
- UIの最適化
- 計算ロジックの改善

## 苦労した点・改善
- GPS誤差への対応
- 予測精度の安定化

## 精度限界と設計上の判断
本アプリでは、同一ルートを1日1回程度しか通れない前提があり、
取得できるデータ数が限られます。
そのため、周期がわずかにずれるだけでも誤差が日単位で増幅し、
翌日には大きなズレとして現れる課題がありました。

この課題から、少量データで「小数点以下まで確定的に当てる」ことには限界があると理解し、
現時点では過度な精度主張を避けています。
今後の改善には、データ収集量を増やす（複数回通行・ユーザー協力など）、
または不確実性を含めて予測する（信頼区間の表示など）設計が必要だと考えています。

## 今後の展望

- EVナビへの応用
